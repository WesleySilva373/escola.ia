# Prompt — Auditoria Final do Projeto

Faça uma auditoria geral do projeto antes da entrega.

Não crie um projeto novo.

Analise o projeto existente e preserve as funcionalidades que já estão funcionando.

Verifique:

- Front-end.
- API de processamento.
- Integração com Gemini.
- Integração com Supabase.
- Upload de arquivos.
- Processamento de PDF.
- Processamento de imagens.
- Processamento de TXT.
- Upload múltiplo.
- Categorização.
- Resumo.
- Tratamento de duplicatas.
- Histórico.
- Consulta de clientes.
- Visualização dos arquivos.
- Segurança das variáveis de ambiente.
- Deploy na Vercel.
- Repositório no GitHub.

Antes de alterar qualquer coisa, identifique o que já funciona.

Faça somente as correções necessárias.

Não crie centenas de arquivos ou uma arquitetura desnecessariamente complexa.

O objetivo é manter o projeto simples, funcional, seguro e pronto para apresentação.
